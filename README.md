# DATABASE DESIGN AND PROTOTYPING PROJECT: DATABASE SYSTEM FOR CONSUMER REVIEW ANALYSIS ON YELP 

## Introduction
This is an individual class project, aiming to design effective data schema based on users requirements and business rules, set up procedures for database administration and set up evaluation plan.

## Dataset Overview 
For this project, the public available user review data of Yelp - consumer reviews platform is utilized.

**Datasets are too large, therefore they are attached via a [Google Drive link](https://drive.google.com/file/d/1x) in the final report, along with SQL statements to create tables, and Python program to insert data.**

Instructions to run program has been uploaded in this repository. 

## Database Design Process

* **Gather users' requirement**s: Including Data requirements, Functional requirements, Potential users, Access Level and Authentication
* **Define major entities and its attributes**: Make sure they followed 1NF (First Normal Form) and Data Integrity, perform professional attribute re-naming if needed
* **Generate Business Rules**: Define entities' relationships and cardinalities
* **Conceptual Data Modeling**: Using Crow Foot Model
* **Conduct thorough Data dictionaries**: Including Attribute name, Data type, Attribute Description, Format, Key, Referenced Table
* **Database Implementaion**: using CREATE TABLE statements in SQL, and Python data insertion programs
* **Perform Sample Queries**
* **Develop Evaluation Plan**
* **Create Procedures for Data Administration activities**
